# Microsoft launches MAI model family at Build 2026 — cutting the OpenAI cord

## What Is It
At Microsoft Build 2026 on June 2 in San Francisco, Microsoft launched its own in-house AI model family called MAI (Microsoft AI). The coding model is designed to replace OpenAI's models inside GitHub Copilot, running entirely on Azure without any OpenAI API calls. This is Microsoft's most direct move yet toward AI independence from its $13B OpenAI partner.

## Key Facts & Stats
- Event: Microsoft Build 2026, June 2–3, San Francisco
- MAI coding model benchmark: performs "at or above" Anthropic Claude 3.7 Sonnet on SWE-bench Verified
- Cost advantage: runs at "significantly lower inference cost" than current Copilot (powered by OpenAI)
- MAI-Thinking-1: Microsoft's first reasoning model not trained by distilling another AI's outputs
- MAI-Image-2.5: ranked #3 in Text-to-Image Arena with score of 1,254 — +72 points vs previous version
- MAI-Transcribe-1.5: #1 globally on FLEURS Word Error Rate benchmark across 25 languages — beats OpenAI Whisper
- MAI-Voice-2: multilingual TTS model covering 15+ languages
- GitHub Copilot shift: moved from flat-fee to AI Credits billing on June 1, 2026 ($0.01/credit)
- Copilot Pro users get $10/month in credits; Pro+ users get $39/month
- Old model: a complex session cost ~$0.08. New model: same session costs $2–$13

## Why It Matters for ZAVINO's Audience
Microsoft invested $13 billion in OpenAI — and is now building models to replace it. This is the tech equivalent of a restaurant chain that invested in a food supplier, then decided to grow its own ingredients because it got too expensive. GitHub Copilot is used by millions of developers worldwide. This shift affects every developer using AI coding tools. Persian developers and tech pros should know the tool they use just got a new engine under the hood — and the pricing model changed overnight.

## The Surprising Angle
The most counterintuitive part: Microsoft poured $13 billion into OpenAI but is now actively building to *not need* OpenAI. The OpenAI dependency created uncomfortable unit economics — meaning Microsoft was losing money on Copilot at scale. The new MAI coding model doesn't call OpenAI at all. It's Microsoft's declaration of AI independence, happening at its own developer conference.
