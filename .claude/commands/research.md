# Research

Deep-dive a topic and save structured findings to the research vault. Findings become source material for future posts.

## Variables

topic: $ARGUMENTS (topic to research — e.g. "Google NotebookLM update", "AI agents in marketing", "Perplexity vs ChatGPT")

---

## Instructions

### Step 1: Check Existing Research

Before starting, search `research/` for any existing files on this topic. If relevant files exist, read them to avoid duplication and build on what's already there.

### Step 2: Research the Topic

Investigate deeply:
- What happened / what is this?
- Key numbers, stats, or data points
- Why does this matter to ZAVINO's audience?
- What's surprising or counterintuitive?
- What can someone DO with this information?
- What's the MENA / Persian-speaking angle, if any?

### Step 3: Identify Content Angles

From the research, extract 3–5 potential post angles:

| Angle | Hook idea | Best format |
|---|---|---|
| Surprising stat | ... | Single post |
| How-to use it | ... | Carousel |
| Before/after | ... | Thread |

### Step 4: Save to Research Vault

Save the research note to the correct subfolder:
- `research/ai-tools/` — AI tools and products
- `research/trends/` — Industry trends and movements
- `research/competitors/` — What other Persian tech pages are doing
- `research/inbox/` — If unsure where it belongs

Use the Research Note Template below.

### Step 5: Update Backlog

Add the best content angle(s) to `context/backlog.md` as new ideas.

### Step 6: Report

1. Summary of key findings (3–5 bullet points)
2. Best post angle identified
3. File saved at: `research/[subfolder]/YYYY-MM-DD-{slug}.md`
4. Ideas added to backlog: yes/no
