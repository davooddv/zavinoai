# Weekly Review

End-of-week reflection and next-week planning. Keeps the content engine sharp and compounds learning over time.

---

## Instructions

### Phase 1: Review the Week

Read all daily notes from this week:
- `context/daily/YYYY-MM-DD.md` × 7 (or however many exist)

Then scan:
- `outputs/instagram/` — what was published this week
- `outputs/twitter/` — what was published this week
- `context/current-data.md` — latest metrics

### Phase 2: Identify Patterns

Answer these questions:

1. **What got published this week?** List all outputs.
2. **What's still in draft?** What got stuck and why?
3. **Which pillar got the most coverage?** (AI / marketing / tech)
4. **Which pillar was neglected?**
5. **Any ideas that came up but weren't captured?**
6. **Did anything in the news/research deserve a post but got missed?**

### Phase 3: Update the Vault

1. **Update `context/current-data.md`** — fill in any new metrics or trending topics discovered this week
2. **Clean up `content/backlog/`** — archive dead ideas, promote the best ones
3. **Move any finished drafts** from `content/posts/` to `outputs/`

### Phase 4: Plan Next Week

Based on the review, create a plan for the coming week:

```markdown
## Week of YYYY-MM-DD

### Content Plan
| Day | Platform | Topic | Format | Status |
|---|---|---|---|---|
| Mon | Instagram | [topic] | Single post | Planned |
| Tue | Twitter | [topic] | Thread | Planned |
| Wed | Instagram | [topic] | Carousel | Planned |
| Thu | Twitter | [topic] | Single | Planned |
| Fri | Instagram | [topic] | Reel script | Planned |

### Research to Do
- [ ] [Topic to research]
- [ ] [Topic to research]

### Focus for This Week
[One sentence: what's the main theme or push this week?]
```

Save this plan to `plans/week-YYYY-MM-DD.md`.

### Phase 5: Report

Deliver a clean weekly summary:
1. Posts published this week: [count]
2. Best performing (if data available)
3. Pillar balance
4. Top 3 ideas for next week
5. Any patterns worth turning into a standing skill or rule
